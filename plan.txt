Goal

SUBSCRIPTION OPTIONS — Phase 0
===============================

Phase 0 objective (Subscription)
- Make all existing `subscription:<key>` gates in `routes/vendor.php` enforceable by mapping them to real
  `store_subscriptions` columns, with matching columns in `subscription_packages` and a consistent copy step
  during purchase/renew/switch.

Permission key inventory (from `routes/vendor.php`)
- Enforced today in middleware:
  - reviews, pos, deliveryman, chat
- Present in routes but NOT enforced today:
  - category, role, deliveryman_list, employee, item, banner, campaign, wallet, wallet_method, coupon,
    advertisement, advertisement_list, addon, store_setup, notification_setup, profile, my_shop,
    expense_report, disbursement_report, vat_report

LOCKED mapping: `subscription:<key>` -> `store_subscriptions.<column>`
A) Already exists (no new DB columns needed)
- subscription:reviews        -> store_subscriptions.review
- subscription:pos            -> store_subscriptions.pos
- subscription:deliveryman    -> store_subscriptions.self_delivery
- subscription:chat           -> store_subscriptions.chat

B) Requires NEW boolean columns (recommended 1:1 naming)
- subscription:category            -> store_subscriptions.category
- subscription:role                -> store_subscriptions.role
- subscription:deliveryman_list    -> store_subscriptions.deliveryman_list
- subscription:employee            -> store_subscriptions.employee
- subscription:item                -> store_subscriptions.item
- subscription:banner              -> store_subscriptions.banner
- subscription:campaign            -> store_subscriptions.campaign
- subscription:wallet              -> store_subscriptions.wallet
- subscription:wallet_method       -> store_subscriptions.wallet_method
- subscription:coupon              -> store_subscriptions.coupon
- subscription:advertisement       -> store_subscriptions.advertisement
- subscription:advertisement_list  -> store_subscriptions.advertisement_list
- subscription:addon               -> store_subscriptions.addon
- subscription:store_setup         -> store_subscriptions.store_setup
- subscription:notification_setup  -> store_subscriptions.notification_setup
- subscription:profile             -> store_subscriptions.profile
- subscription:my_shop             -> store_subscriptions.my_shop
- subscription:expense_report      -> store_subscriptions.expense_report
- subscription:disbursement_report -> store_subscriptions.disbursement_report
- subscription:vat_report          -> store_subscriptions.vat_report

`subscription_packages` must mirror the same fields
- Add the SAME boolean columns to `subscription_packages` so Admin can configure per package:
  - category, role, deliveryman_list, employee, item, banner, campaign, wallet, wallet_method, coupon,
    advertisement, advertisement_list, addon, store_setup, notification_setup, profile, my_shop,
    expense_report, disbursement_report, vat_report

Rollout defaults (choose one)
- Safe rollout (no vendor break): new columns default TRUE
- Strict rollout (admin must enable): new columns default FALSE

Middleware unknown-key behavior (choose one)
- Safer: unknown subscription:<key> => DENY with message
- Non-breaking rollout: unknown subscription:<key> => ALLOW (not recommended long-term)

Status (Implemented)
- DB columns added to `subscription_packages` and `store_subscriptions` with default ON (safe rollout).
- Admin package create/edit supports all permissions.
- Subscription purchase/renew/switch copies permission snapshot into `store_subscriptions`.
- Vendor access is enforced for all `subscription:<key>` via `app/Http/Middleware/Subscription.php`.
- Vendor sidebar hides menus based on subscription permissions (UI matches enforcement).
- Admin and Vendor “Package Overview/Details” screens display the new permissions.

Maintenance / Next Steps
- When you add or rename any `subscription:<key>` in routes, update:
  1) `app/Http/Middleware/Subscription.php` permission map
  2) DB columns in both tables (`subscription_packages`, `store_subscriptions`)
  3) Snapshot copy in `Helpers::subscription_plan_chosen()`
  4) Sidebar hide rules (optional but recommended)
- Run the audit command to confirm everything is wired:
  - `php artisan subscription:audit`
